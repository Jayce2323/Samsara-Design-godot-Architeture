			*Item Composition System*

CORE STRUCTURE
	Items are atomic objects that exist independently. 
	Items can be combined with other Items to create composite Items. 
	Composite Items reference their component Items via parts array. 
	All three (component A, component B, composite) are simultaneous Items.

ITEM HIERARCHY
	Items exist at single level.  No "equippable only" vs "material only" distinction.
	An Item is defined by: 
		- Category (what it is conceptually)
		- SubCategory (what role it serves)
		- TypeCategory (specific variant)
		- material_type (what it's made of)
		- mass (weight)
		- parts (Array of other Items that compose it, if composite)

COMPOSITION
	parts: Array[ItemTypeSubCategory]
		References to other Items that compose this Item. 
		Example:   SWORD. parts = [CROSSGUARDHILT, DOUBLE_EDGE_BLADE]
		Meaning: This SWORD is made of a CROSSGUARDHILT item and a DOUBLE_EDGE_BLADE item. 
	
	When parts array is empty:  Item is atomic/component. 
	When parts array contains items:  Item is composite. 
	Item can be both:  A composite sword that is itself a component in something larger.

PROPERTY AGGREGATION
	Composite Item properties = derived from component Item properties.
	mass:  sum of all component masses
	material_type: array of all component materials
	durability/hardness: aggregate based on weakest/strongest component
	damage values: aggregate based on component composition
	
	System reads component Item data, calculates composite properties. 
	No separate property storage for compositesâ€”always derives from components. 

CRAFTING FLOW
	1. Player selects Item A (HILT) 
	2. Player selects Item B (BLADE)
	3. Craft method validates combination (if validation exists)
	4. New Item C created with: 
		- TypeCategory = result type (SWORD)
		- parts = [reference to Item A, reference to Item B]
		- material_type = [A. material_type, B.material_type]
		- mass = A.mass + B.mass
		- properties calculated from A + B properties
	5. Item A and Item B continue to exist as independent Items
	6. Item C is new Item that references them

USAGE
	Any Item can be: 
		- Used standalone (HILT equipped as weapon, BLADE as knife, SWORD as sword)
		- Used in crafting as component
		- Stored in inventory
		- Traded/dropped
		- Consumed/destroyed
	
	System processes mechanics based on Item being used, regardless of composition state.
